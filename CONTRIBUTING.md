# How to Contribute

First off, thank you for considering contributing! This project is stronger with your input.

### Reporting Bugs

* **Ensure the bug was not already reported** by searching in the [Issues](https://github.com/your-username/your-repo/issues) section.
* If you're unable to find an open issue addressing the problem, **open a new one**. Be sure to include:
    1.  A **clear and descriptive title**.
    2.  The **script name and version** you are using.
    3.  **Steps to reproduce** the bug.
    4.  **What you expected to happen** vs. **what actually happened**.
    5.  **Screenshots** if possible, especially for visual bugs on a chart.

### Suggesting Enhancements

* **Open a new issue** with the title `[Feature Request] Your Feature Name`.
* Provide a detailed description of the proposed feature and why it would be beneficial.

### Pull Requests

This is the best way to contribute your own code, fixes, or strategy ideas.

1.  **Fork the Repository:** Click the 'Fork' button at the top right of this page.
2.  **Create a Branch:** Create a new branch for your changes:
    ```sh
    git checkout -b feature/my-new-strategy
    ```
    or
    ```sh
    git checkout -b fix/my-bug-fix
    ```
3.  **Make Your Changes:** Write your code. Please ensure it is well-commented, especially for complex logic.
4.  **Update CHANGELOG.md:** If your change is a bug fix, feature, or modification, add an entry to the `[Unreleased]` section of the `CHANGELOG.md`.
5.  **Commit Your Changes:** Use a clear and descriptive commit message.
    ```sh
    git commit -m "Add: My New Strategy logic"
    ```
6.  **Push to Your Branch:**
    ```sh
    git push origin feature/my-new-strategy
    ```
7.  **Open a Pull Request:** Go to the original repository and you will see a prompt to create a Pull Request from your forked branch. Please provide a clear description of your changes in the PR.
